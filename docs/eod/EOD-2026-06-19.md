  # EOD — Friday, June 19, 2026

## What I worked on today

Implemented the full **Snapshots** feature for the Church Admin desktop application.

### Models
- Created `SnapshotType` enum (`AUTO_DAILY`, `PRE_IMPORT`, `MANUAL`) in `com.churchadmin.models.enums`.
- Created `SnapshotEntry` — a lightweight disk descriptor (not a JPA entity) with `filename`, `createdAt`, `type`, `sizeBytes`, plus helper methods `getFormattedSize()` and `getTypeLabel()`.
- Created `SnapshotPayload` — the Jackson-serialized JSON envelope (`schemaVersion`, `exportedAt`, `exportedBy`, `snapshotType`, `appVersion`, and raw `List<Map<String, Object>>` per table).

### Services
- Rewrote `SnapshotService` from scratch using `JdbcTemplate` and Jackson with `JavaTimeModule`. Key behaviours:
  - Snapshots are structured JSON files (not raw `.db` copies), stored at `~/.churchadmin/snapshots/`.
  - Filename format: `yyyy-MM-dd_HH-mm-ss_TYPE.json`; imported external files get an `IMPORTED_` prefix.
  - `listSnapshots()` uses a regex to parse filenames; non-conforming files are skipped with a warning.
  - `restoreSnapshot()` is `@Transactional`: creates a PRE_IMPORT safety backup, deletes all rows in reverse FK order (`transactions → members → transaction_categories`), then re-inserts from the payload in FK-safe order — all-or-nothing.
  - `enforceRetention()` auto-prunes AUTO_DAILY (keep 30) and PRE_IMPORT (keep 10); MANUAL snapshots are never deleted.
- Created `StartupSnapshotJob` (`@Component`, `@PostConstruct`) — starts a daemon thread on Spring boot that creates an AUTO_DAILY snapshot at most once per calendar day; failures are logged as warnings and never surface to the user.

### Controller & UI
- Rewrote `SnapshotsController` with a full JavaFX table: Date/Time, Type, Size, and an Actions column with per-row **Export** / **Restore** / **Delete** buttons.
- All `SnapshotService` calls run on daemon background threads via `javafx.concurrent.Task`, with results delivered back to the FX thread. A double-submit guard disables toolbar buttons for the duration of each operation.
- Rewrote `Snapshots.fxml`: header bar with title, subtitle, and toolbar buttons; `TableView` with `VBox.vgrow="ALWAYS"`; footer status bar with a `ProgressIndicator` and a status label that switches between `.status-ok` and `.status-error` CSS states.

### Configuration & Infrastructure
- Updated `application.properties`: replaced `app.snapshots.*` keys with `snapshots.dir`, `snapshots.retention.auto-daily`, and `snapshots.retention.pre-import`.
- Added new CSS classes to `app.css`: `.screen-root`, `.screen-header`, `.screen-title`, `.screen-subtitle`, `.data-table`, `.table-placeholder`, `.status-bar`, `.status-label` (with `.status-ok` / `.status-error` modifiers), `.small-spinner`, and `:disabled` variants for all three button styles.
- Added ~30 `snapshots.*` i18n keys to all three locale files (English, Dutch, Arabic).
- Removed the old `createDailySnapshotIfNeeded()` call from `ChurchAdminApplication` — `StartupSnapshotJob` now owns that responsibility.
- Updated `ImportService` to call the new `createSnapshot(PRE_IMPORT, …)` API in place of the removed `createPreImportSnapshot()` method.
