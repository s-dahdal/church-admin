# EOD — Tuesday, June 16, 2026

## What I worked on today

- Diagnosed and fixed multiple startup failures in the Church Admin desktop application
  (Spring Boot + JavaFX + SQLite).
- Resolved a Java version mismatch: the system default was Java 25, but the project
  requires Java 21. Configured the build to use the correct JDK.
- Fixed a Spring Data JPA repository scanning issue caused by the main application class
  sitting in `com.churchadmin.app` rather than the root `com.churchadmin` package.
  Spring Boot's `@AutoConfigurationPackage` was scoping the JPA repository scan to
  the wrong subpackage, so no repositories were registered. Added `@EnableJpaRepositories`
  and `@EntityScan` annotations to explicitly target the correct packages.
- Resolved a Hibernate schema validation error: SQLite stores all date/time columns as
  TEXT, but Hibernate expected TIMESTAMP. Changed `ddl-auto` from `validate` to `none`,
  delegating full schema ownership to Flyway as intended.
- Added the missing `dashboard.title` i18n key to all three locale files (English, Dutch,
  Arabic), which was causing the dashboard screen to fail to load on startup.
- Created an IntelliJ IDEA run configuration (`Church Admin`) so the application can be
  launched directly from the IDE with the correct `JAVA_HOME` set, without needing to
  run Maven manually from the terminal.
