### Bug fixes

- **JPQL startup failure**: removed `CASE WHEN t.type = 'INCOME'` and `sumByTypeAndPeriod` `@Query` methods from `TransactionRepository` — comparing enum fields to string literals fails Spring Data's validation at startup. All balance calculations moved to Java streams in `FinancialService`.
- **Empty transaction table**: `Transaction.category` and `Transaction.member` are `@ManyToOne(fetch = LAZY)`. After the Hibernate session closes, JavaFX cell factories accessing these proxies threw `LazyInitializationException` silently, rendering the table blank. Fixed by adding `LEFT JOIN FETCH` variants (`findAllEager`, `findByMemberIdEager`, `findTop10EagerOrderByDateDesc`) so associations are loaded in a single query.
