# EOD — Wednesday, June 24, 2026

## What I worked on today

Continued building and refining the **Smart Snapshot Merge Import** feature for the Church Admin desktop application.

**Revised the core merge model (v2):**
- Replaced the old timestamp-based CONFLICT/UPDATABLE classification with a simpler entity-specific approach: members are NEW, IDENTICAL, or PENDING; transactions are deduplicated by composite business-field key (NEW or IDENTICAL); categories are completely ignored.
- Replaced `ConflictRecord` with `PendingMemberRecord` (local record, incoming record, changed fields, resolved flag).
- Rewrote `MergeImportReport` with per-entity counters and a `pendingMembers` list.

**Rewrote `MergeImportService`:**
- `analyze()` — members matched by UUID, transactions matched by composite key (date + amount + type + category_id + description + member_id), categories skipped entirely.
- `applyAutomatic()` — inserts new members and new transactions in one transaction, preceded by a PRE_IMPORT safety snapshot.
- `applyPendingMember()` / `discardPendingMember()` — admin-driven per-record resolution with no side effects on transactions (removed the deferred transaction concept entirely after realising PENDING members already exist locally, so their FK is always valid).

**Rewrote `MergeImportReviewController` and `MergeImportReview.fxml`:**
- Replaced the three-tab conflict layout with a single pending-members `TableView`.
- Summary bar shows: New Members, New Transactions, Skipped, Pending Review.
- Per-row Keep Local / Accept Incoming buttons; bulk Accept All / Keep All actions.
- Renamed the Apply button to **Add New Records** to better describe what it does.

**Added Done-button guard:**
- Clicking Done now checks whether new records have been applied (`newRecordsApplied` flag) and whether unresolved pending members remain.
- If either condition is true, a confirmation dialog appears with a specific message listing the outstanding counts (e.g. "3 new record(s) have not been added yet. 2 member(s) are still pending review.") followed by a "If you leave now, these changes will not be applied. Continue?" prompt.

**Made transactions read-only:**
- Existing transactions in the Transactions screen can no longer be edited — the detail form is now read-only when opened for an existing record, with all fields disabled and the Save button hidden. Delete remains available.

**i18n:**
- Updated all `merge.*` keys in EN, NL, and AR across all three locale files to match the new model and button labels.
