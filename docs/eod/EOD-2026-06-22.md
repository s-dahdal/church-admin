### AWT Classloader Crash Fix

`Taskbar.getTaskbar()` initializes the AWT Toolkit on the JavaFX Application Thread as a side-effect, which **nulls out** the thread's context classloader. This caused every lazy `FXMLLoader.load()` call to throw:

```
NullPointerException: Cannot invoke "ClassLoader.loadClass(String)"
because the return value of "FXMLLoader.getClassLoader()" is null
```

Two-layer fix applied:

1. **`ChurchAdminApplication.start()`** — capture the classloader before the Taskbar call and restore it immediately after.
2. **`JavaFXConfig.FXMLLoaderFactory`** — explicitly call `loader.setClassLoader(applicationContext.getClassLoader())` on every `FXMLLoader` instance so it never falls back to the (potentially null) thread context classloader.
